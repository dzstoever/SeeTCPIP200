SeeTCPIP Fact Sheet(release 1.5.9)

Product Description

SeeTCPIP provides a detailed view and analysis tool for your TCP/IP stack(s) running on VSE. The product allows you to watch network activity on your systems in real-time and produces historical charts and reports. It allows you to see what applications, interfaces, hosts, foreign IP addresses, and connections are consuming the most system resources. SeeTCPIP also provides an interactive command interface. It provides a rich, simple user-interface allowing users with varied expertise to benefit and successfully monitor your mission critical mainframe operation.

Features

Monitor Highlights

  • Network activity/efficiency and client counts filtered by protocol/port (IP,TCP,UDP,ICMP,Telnet,FTP,HTTP,LP,GPS).
  • Misrouted, discarded, and unsupported packets
  • Checksum and datagram length errors
  • Rejected connections
  • Individual connections filtered by foreign ip, BSDC socket connection, CICS transaction, inbound/outbound byte counts, retransmits, port, PID, and more.
  • FTP users, sessions, and transactions.
  • TCP/IP Dispatcher including dispatch counts, time, waits and storage releases.
  • Turbo Dispatcher CPU activity including Job/Step information and SIO counts.

TCP/IP Command Console

  • Issue commands directly to TCPIP.
  • View console messages in real-time and historically.
  • Provides interactive help and feedback on all CSI TCP/IP commands and messages.

Technical Details

  • (Mainframe) Must be running CSI TCPIP 1.5E.
  • (PC) Must be running Windows 2000, XP, or Vista(testing in progress).
  • (PC) Stores all data and handles all user interaction, thus eliminating most overhead from the customers mainframe system.

Release Notes

The user-interface has been significantly redesigned.

The product has been divided into two components.

  1. Network communications and data collection are run and controlled silently as a "Windows Service".
  2. The user-interface is a completely separate component that runs as a "Windows Application".

This allows the user to set up all of the TCP/IP systems that they are interested in and then start the "Windows Service" once and leave it running. Once this is done, they can open and close the "Windows Application" at any time to view their system(s) activity. So as long as the the "Windows Service" is running, information is collected on all configured systems.
